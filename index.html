A Deep Dive Into the Email Regular Expression, Otherwise Known as a Regex!

/^([a-z0-9_\.-]+)@([\da-z\.-]+)\.([a-z\.]{2,6})$/

1. ^ (Start of String)

This symbol asserts the position at the start of the string. It ensures that the regex matches from the beginning of the input.

2. ([a-z0-9_\.-]+) (Username Part)

This part matches the username portion of the email, which can consist of:

  • [a-z]: Any lowercase letter from 'a' to 'z'
  • [0-9]: Any digit from 0 to 9
  • _: An underscore
  • \.: A dot (escaped with a backslash)
  • -: A hyphen

The + quantifier ensures that this part appears one or more times.

3. @ (At Symbol)

This is a literal match for the "@" symbol, which separates the username from the domain in an email address.

4. ([\da-z\.-]+) (Domain Part)

This part matches the domain name, allowing:

  • \d: Any digit (0-9)
  • [a-z]: Any lowercase letter from 'a' to 'z'
  • \.: A dot
  • -: A hyphen

The + quantifier ensures that this part appears one or more times.

5. \. (Dot Literal)

This is a literal match for the dot that separates the domain name from the top-level domain (TLD).

6. ([a-z\.]{2,6}) (Top-Level Domain Part)

This part matches the top-level domain (TLD), allowing:

  • [a-z]: Any lowercase letter from 'a' to 'z'
  • \.: A dot (optional)

The {2,6} quantifier ensures that the TLD has between 2 and 6 characters (e.g., "com", "org", "co.uk").

7. $ (End of String)

This symbol asserts the position at the end of the string, ensuring that the regex matches until the very end of the input.
